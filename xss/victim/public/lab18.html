Angular — bypassSecurityTrustHtml

Angular sanitizes HTML bound to the DOM by default. But a developer used bypassSecurityTrustHtml() to explicitly mark attacker input as "trusted" — defeating all protection.

📜Vulnerable Angular Code
// Angular component — TypeScript
constructor(private sanitizer: DomSanitizer) {}

setUserBio(input: string) {
  // Safe: Angular sanitizes by default
  this.safeBio = input;  // used with [innerHTML] → sanitized

  // BUG: explicitly trusting attacker input
  this.trustedBio = this.sanitizer
    .bypassSecurityTrustHtml(input);  // ← marks as trusted
}

<!-- Template -->
<div [innerHTML]="safeBio"></div>        <!-- sanitized -->
<div [innerHTML]="trustedBio"></div>     <!-- trusted = raw -->
📚Interview Prep: Angular Security
  • Default: Angular sanitizes all values bound via [innerHTML], [style], [src], [href].
  • Bypass: bypassSecurityTrustHtml(), bypassSecurityTrustUrl(), bypassSecurityTrustResourceUrl(), bypassSecurityTrustStyle().
  • AngularJS legacy: Client-side template injection via {{constructor.constructor('code')()}} sandbox escapes — historical but still asked in interviews.
  • Fix: Never call bypassSecurityTrust* with user input. If rich text is needed, sanitize with DOMPurify before bypassing.
