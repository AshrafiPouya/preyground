Vue — v-html Directive

Vue's {{ }} interpolation is safe — it renders as text. But v-html renders raw HTML, bypassing Vue's built-in XSS protection.

●Vue Post Component

✓ Safe — Vue {{ }} interpolation (auto-escaped):

{{ submitted }}

✗ Unsafe — v-html directive (raw HTML):

📜Vulnerable Vue Code
<!-- Safe: Vue escapes text interpolation -->
<p>{{ userContent }}</p>

<!-- BUG: v-html renders raw HTML -->
<div v-html="userContent"></div>

Fix: Use {{ }} for text content. If you need rich text, sanitize with DOMPurify before passing to v-html.

Hints
#1{{ }} vs v-html
Vue's {{ }} syntax renders the value as text — it's always safe. v-html renders as raw HTML — it's the escape hatch, and it's dangerous with untrusted input.
#2innerHTML quirk applies
v-html works by setting el.innerHTML. The same rules apply: <script> won't run, but onerror, onload, <svg onload=...> will.
#3The payload
Type <img src=x onerror=captureFlag()> in the content field. Watch how Vue's safe output shows it as text, but v-html executes it.
