DOM XSS — postMessage → innerHTML

A message event listener passes event.data directly into innerHTML without checking event.origin. Any window can send a malicious payload.

📩Live Notification Widget

This widget receives real-time updates via postMessage:

Waiting for messages…
📜Vulnerable Code
// BUG: no origin check, unsafe innerHTML sink
window.addEventListener('message', e => {
  // Should check: if (e.origin !== 'http://localhost:3000') return;
  document.getElementById('notification-area').innerHTML = e.data;
});

Fix: Always validate event.origin against a known-good allowlist, AND sanitize event.data before inserting into the DOM.

🎯Exploit Flow

From the attacker origin, open or iframe this page, then call:

victimWindow.postMessage('<img src=x onerror=captureFlag()>', '*');
