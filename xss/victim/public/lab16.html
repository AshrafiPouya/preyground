React — dangerouslySetInnerHTML

React auto-escapes {value} interpolation, but dangerouslySetInnerHTML opts out entirely. This component passes attacker-controlled bio text straight through.

⊙React User Profile Component
📜Vulnerable React Code
// ProfileCard component — VULNERABLE
function ProfileCard({ bio }) {
  return (
    <div className="profile">
      <h3>User Profile</h3>
      {/* Safe: React auto-escapes {bio} */}
      <p>{bio}</p>

      {/* BUG: dangerouslySetInnerHTML bypasses React's escaping */}
      <div dangerouslySetInnerHTML={{ __html: bio }} />
    </div>
  );
}

Key: {bio} is safe — React HTML-encodes it. dangerouslySetInnerHTML skips this and injects raw HTML.

Hints
#1React is safe by default
React escapes all JSX interpolated values by default. To get XSS, find where the developer turned off that protection.
#2dangerouslySetInnerHTML
The "dangerously" prefix is React's warning. This prop sets raw innerHTML — any HTML you put in the bio becomes executable.
#3The payload
Enter <img src=x onerror=captureFlag()> in the bio field. Notice the safe {bio} paragraph shows it as text — but dangerouslySetInnerHTML executes it.
