💡XSS — The Core Idea

XSS happens when an application includes attacker-controlled data in a page in a way that causes the browser to execute attacker script in a victim's session.

The three roots: reflected (echo from URL), stored (persisted server-side), DOM-based (client-side sink). The craft is in context (HTML body / attribute / JS string / URL / CSS) and bypass (defeating filters and CSP).

Reflected

Input from request echoed into response — delivered via crafted link.

Stored

Payload persisted server-side; fires for every viewer. Generally higher impact.

DOM-based

Entirely client-side: source → sink in JS. Server never sees the payload.
