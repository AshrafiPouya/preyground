Schema Enumeration

Knowing information_schema by name impresses interviewers. Know the cross-DB variants.

The 30-Second Answer

"Once you have SQL injection, you almost never know the table and column names up front. So you query the database's own metadata — information_schema in MySQL/PostgreSQL/MSSQL, sqlite_master in SQLite, ALL_TABLES in Oracle — to enumerate every table and column, then extract from the one containing sensitive data."

Common Interview Questions

Query information_schema.tables for table names, then information_schema.columns filtered by table_name for column names. Use GROUP_CONCAT to dump many rows at once: SELECT GROUP_CONCAT(table_name) FROM information_schema.tables WHERE table_schema=database(). In SQLite: SELECT GROUP_CONCAT(name) FROM sqlite_master WHERE type='table'.
information_schema is a virtual database (or schema) present in MySQL, PostgreSQL, and MSSQL that contains metadata about all databases, tables, columns, and their properties — data types, character sets, privileges. It's read-only and always accessible to any connected user (unless privileges are restricted). Knowing it's universally available and queryable via SQL is the key insight.
MySQL/PostgreSQL/MSSQL: information_schema.tables and information_schema.columns. SQLite: sqlite_master (table name, SQL CREATE statement). Oracle: ALL_TABLES, ALL_COLUMNS, ALL_TAB_COLUMNS — no information_schema. Postgres also has pg_catalog as an alternative. Knowing these by name is what separates a practiced tester from someone who only knows theory.

Explain It to a Developer (the Fix)

You can't hide the schema from a successful injection — it's by design part of the database. The fix is upstream: parameterized queries prevent injection, and least-privilege accounts can limit which tables are readable even if injection occurs.

One-Liners to Remember

SELECT table_name FROM information_schema.tables WHERE table_schema=database()
SELECT column_name FROM information_schema.columns WHERE table_name='users'
SELECT GROUP_CONCAT(table_name) FROM information_schema.tables-- (dump all at once)
SELECT name FROM sqlite_master WHERE type='table'-- (SQLite)
SELECT table_name FROM ALL_TABLES-- (Oracle)

Red Flags to Avoid Saying

❌ Assuming every database has information_schema — Oracle doesn't; SQLite uses sqlite_master.
❌ Not knowing GROUP_CONCAT or its cross-DB equivalents (STRING_AGG in PostgreSQL/MSSQL).
❌ Thinking schema is hidden from injection — it's always accessible via metadata tables.
