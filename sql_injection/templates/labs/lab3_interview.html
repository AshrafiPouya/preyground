UNION-Based SQL Injection

Expect to whiteboard the column-counting workflow. Know ORDER BY vs NULL probing cold.

The 30-Second Answer

"UNION-based SQLi appends a second SELECT statement to the original query, pulling data from other tables into the reflected results. It requires matching column count and compatible data types. The workflow is: count columns → find reflected columns → UNION SELECT the target data."

Common Interview Questions

Two methods: (1) ORDER BY — inject ORDER BY 1--, then ORDER BY 2--, incrementing until you get an error. The last successful number is the column count. (2) UNION SELECT NULL — add NULLs one at a time: UNION SELECT NULL--, UNION SELECT NULL,NULL-- until no error. NULL is type-compatible with everything. ORDER BY is often cleaner and less disruptive to the query structure.
If a column is numeric, putting a string there causes a type error. Use NULL in numeric columns and put your extracted string in a text-compatible column. Replace NULLs one at a time: UNION SELECT 'test',NULL,NULL-- — if 'test' appears, column 1 is a string column. Rotate until you find a usable reflected column.
Replace each NULL with a unique distinguishable string: UNION SELECT 'col1marker',NULL,NULL--. Look for 'col1marker' in the HTML response. Repeat for each position. Once you know which positions are reflected, put your real payload (subquery) there.
Use GROUP_CONCAT() (MySQL) or STRING_AGG() (PostgreSQL) to concatenate all values from a column into a single string: UNION SELECT GROUP_CONCAT(username,':',password SEPARATOR '|'),NULL FROM users--. SQLite uses GROUP_CONCAT(col, ',').

Explain It to a Developer (the Fix)

Parameterized queries prevent UNION injection entirely — the UNION keyword can never be injected because the query structure is fixed. Additionally, apply least-privilege database accounts so even a successful injection can't read sensitive tables.

Gotchas & Follow-Ups

ORDER BY is cleaner for initial column counting — it doesn't change the returned data, just triggers errors at the boundary. UNION SELECT NULL is more flexible once you've confirmed the number and need to probe types. In practice: ORDER BY to count, then NULL probing to find usable columns.
UNION extends a single SELECT — it appends a second SELECT and merges result sets. Stacked queries (using ;) run an entirely separate SQL statement. Stacked queries require the DB driver to support multi-statement execution (MySQL's mysqli vs PDO handle this differently). UNION is more universally exploitable for data extraction.

One-Liners to Remember

' ORDER BY 1--
' UNION SELECT NULL,NULL,NULL--
%' UNION SELECT table_name,NULL,NULL FROM information_schema.tables--
%' UNION SELECT GROUP_CONCAT(username,0x3a,password),NULL FROM users--

Red Flags to Avoid Saying

❌ Confusing UNION-based with stacked queries — they are fundamentally different mechanics.
❌ Forgetting type compatibility — saying "just put your payload in any column" without addressing numeric columns.
❌ Only knowing the column-counting theory without being able to describe how to find reflected columns.
