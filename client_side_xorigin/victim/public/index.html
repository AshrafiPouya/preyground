🔒 Same-Origin Policy — Quick Reference

An origin is the combination of scheme + host + port. Two URLs share an origin only when all three components match exactly. The browser's Same-Origin Policy prevents JavaScript on one origin from reading responses from a different origin — unless the server explicitly grants permission.

URL A URL B Same origin? Reason
http://localhost:3000/foo http://localhost:3000/bar ✓ Yes Same scheme, host, port
http://localhost:3000 http://localhost:3001 ✗ No Different port
http://localhost:3000 https://localhost:3000 ✗ No Different scheme
http://app.example.com http://api.example.com ✗ No Different subdomain

SOP Blocks (cross-origin JS cannot…)

  • Read fetch() / XHR response bodies
  • Read cross-origin cookies or localStorage
  • Access cross-origin iframe DOM
  • Read WebSocket message data (not enforced by WS spec)

SOP Allows (cross-origin…)

  • Embed images, scripts, CSS, iframes
  • Form submissions (no JS read-back)
  • WebSocket connections (cookies sent!)
  • postMessage to any window (by design)
